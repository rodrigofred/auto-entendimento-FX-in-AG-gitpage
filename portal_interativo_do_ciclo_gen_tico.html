O Ciclo dos Algoritmos Genéticos

Os algoritmos genéticos simulam o processo de seleção natural proposto por Charles Darwin. Uma população de soluções potenciais (indivíduos) evolui ao longo de gerações. Esse processo é controlado por variáveis cruciais: o tamanho da população determina o espaço de busca inicial, a taxa de crossover controla o grau de recombinação de boas características, e a taxa de mutação impede que o algoritmo fique preso em mínimos locais, introduzindo diversidade aleatória.

"Esse ciclo se repete até atingir o critério de parada — fitness ótimo, convergência ou número máximo de iterações/gerações."

O Problema da Mochila Binária (0-1 Knapsack)

Na mochila binária, temos um conjunto de itens, cada um com um peso e um valor. O objetivo é escolher um subconjunto de itens de modo a maximizar o valor total, sem exceder a capacidade máxima de peso suportada pela mochila. No AG, cada gene do cromossomo representa a presença (1) ou ausência (0) de um item específico.

Penalização de Fitness

Caso uma solução ultrapasse a capacidade de peso, o seu fitness é penalizado reduzindo-se o seu valor final proporcionalmente ao excesso, desencorajando a seleção dessa linhagem.

Heurística de Reparo

Mecanismo que varre a solução inválida e remove os itens menos eficientes (pior relação valor/peso) até que o peso total caiba novamente no limite da mochila.

Especificações Técnicas: 4 Formas do Ciclo de Mochila

F1 Texto Linear

[LOG] Início → gerar população (binária: cada gene = incluir item) → decodificar cromossomos → calcular fitness (soma valores; penalizar se peso > capacidade) → seleção (roulette/torneio) → crossover → mutação → reparo/ajuste (garantir capacidade) → avaliar nova população → atualizar melhor solução → repetir até critério de parada.

Variáveis: capacidade, pesos[], valores[], pop_size, taxa_cross, taxa_mut, max_gen, penalidade_peso.

F3 Fluxo Compacto

POP → DECODE → FITNESS → SEL → CROSS → MUT → REPAIR → EVAL → UPDATE_BEST → STOP
Vars: capacity, weights[], values[], pop_size, cross_rate, mut_rate, penalty_factor, gen_limit.

F2 Tabela ASCII Mochila

+----------------------+-------------------------------+ | Etapa | Variáveis / Observações | +----------------------+-------------------------------+ | Representação | cromossomo binário; n itens | | Decodificação | peso_total, valor_total | | Fitness | valor_total - penalidade(peso)| | Seleção | método: torneio/roulette | | Crossover | taxa_cross, ponto_corte | | Mutação | taxa_mut, flip bit | | Reparo/Validação | remover itens até caber | | Substituição | elitismo ou geração completa | | Parada | max_gen ou fitness_target | +----------------------+-------------------------------+

F4 Algoritmo de Execução

inicializar(pop_size) for gen in range(max_gen): for individuo in populacao: peso, valor = decodificar(individuo) if peso <= capacidade: fitness = valor else: fitness = valor - penalidade*(peso - capacidade) selecao = escolher(populacao, fitness, method) filhos = crossover(selecao, taxa_cross) mutados = mutacao(filhos, taxa_mut) para cada filho em mutados: if peso(filho) > capacidade: filho = reparar(filho) # remove itens menos eficientes até caber populacao = substituir(populacao, mutados, elitismo) atualizar(melhor_sol) if melhor_sol.fitness >= fitness_target: break retornar melhor_sol

Modos Alternativos de Execução de AGs

Dependendo do tipo de problema e do espaço de busca, o fluxo clássico do algoritmo genético pode ser modificado. O Elitismo protege as melhores soluções, a Busca Local intensifica a exploração de vizinhanças (Algoritmos Meméticos), e a Mutação Aritmética adapta-se a soluções numéricas contínuas sem necessitar de crossover. Veja abaixo a comparação direta de comportamento.

Matriz Comparativa de Desempenho

Modo Convergência Diversidade Complexidade Melhor uso
Clássico (Crossover + Mutação) Média Média Baixa Problemas gerais, robustez padrão
Mutação Aritmética (Sem Crossover) Rápida Baixa Baixa Representações numéricas ou reais naturais
Steady-state com Elitismo Lenta → Estável Alta Média Preservar boas soluções, evitar descarte de elite
Híbrido / Memético (GA + Busca Local) Rápida/Precisa Média Alta Otimização refinada, combinatórios altamente complexos

Modo 1 — Clássico (Crossover + Mutação)

Vars: pop_size, cross_rate, mut_rate

Algoritmo Genético padrão. Gera a população inicial de forma aleatória, aplica seleção proporcional (como roleta) ou torneio, realiza o cruzamento (crossover) de 1 ponto ou uniforme e aplica a mutação bit-flip em cada gene dos novos gerados com probabilidade definida.

# Log Curto: [LOG] POP -> FIT -> SEL -> CROSS -> MUT -> UPDATE -> REPEAT

Modo 2 — Mutação Aritmética (Sem Crossover)

Vars: pop_size, op_mut, n_bits, bound_policy

Nesta abordagem de mutação intensiva, cromossomos binários são tratados diretamente como valores numéricos inteiros ou de precisão flutuante. O algoritmo aplica operações matemáticas clássicas (incremento, decremento, exponenciação) para produzir variações mutadas, sem a necessidade de crossover.

# Log Curto: [LOG] POP -> DECOD -> ARITH_MUT -> ENCODE -> EVAL -> REPEAT

Modo 3 — Steady-state com Elitismo

Vars: pop_size, offspring_rate, elitism_count

Em vez de substituir todos os indivíduos de uma geração de uma vez, esta variante gera apenas um pequeno lote de filhos a cada iteração (offspring_per_gen), substituindo diretamente as piores soluções da geração anterior e preservando rigidamente a fração de elite intacta.

# Log Curto: [LOG] INIT -> EVAL -> (SEL->CROSS->MUT->EVAL->REPLACE) -> KEEP_ELITE

Modo 4 — Híbrido / Memético (GA + Busca Local)

Vars: pop_size, cross_rate, ls_rate, ls_iters

Agrega o poder global de exploração dos AGs clássicos com a velocidade de refino local de algoritmos de subida de encosta (Hill-Climbers) ou busca tabu. Aplica-se busca local em soluções promissoras geradas para acelerar a precisão em problemas complexos.

# Log Curto: [LOG] POP -> FIT -> SEL -> CROSS -> MUT -> LOCAL_SEARCH -> UPDATE -> REPEAT
